Нашёл проблему. У тебя в hero-секции лежат **два** одинаковых медиа-элемента друг на друге, и оба с `opacity-40`:

1. `<img class="hero-fallback ... opacity-40">` — фолбэк-картинка
2. `<video class="hero-video ... opacity-40">` — видео поверх неё

Когда видео **успешно** загружается и играет (а оно у тебя играет — `readyState: 4`, `paused: false`), фолбэк **не скрывается**. Оба слоя с `opacity-40` накладываются друг на друга, и через них ещё просвечивает `bg-gray-900` секции. Плюс поверх обоих лежит градиентный оверлей `from-gray-950/95 via-gray-900/80 to-gray-900/20`. В итоге фон выглядит мутно и задвоенно — слева почти чёрное, справа чуть видно крышу, но всё замылено.

**Фикс — скрывать фолбэк, когда видео начинает играть.** В твоём скрипте в конце файла ты уже обрабатываешь `heroVideo`, но только на случай **фейла** (прячешь видео). Нужно зеркальную логику — при **успешном** плее прятать фолбэк:

```js
const heroVideo = document.querySelector('.hero-video');
const heroFallback = document.querySelector('.hero-fallback');

if (heroVideo) {
  heroVideo.muted = true;

  const tryPlay = () => {
    heroVideo.play()
      .then(() => {
        // ✅ Видео пошло — прячем фолбэк
        if (heroFallback) heroFallback.style.display = 'none';
      })
      .catch(() => {
        // ❌ Видео не играет — прячем видео, фолбэк остаётся
        heroVideo.style.display = 'none';
      });
  };

  if (heroVideo.readyState >= 2) tryPlay();
  else heroVideo.addEventListener('canplay', tryPlay, { once: true });

  // Safety timeout — если видео так и не стартовало, убираем его
  setTimeout(() => {
    if (heroVideo.paused && heroVideo.currentTime === 0) {
      heroVideo.style.display = 'none';
    }
  }, 5000);
}
```

Ключевое изменение — `.then(() => { heroFallback.style.display = 'none'; })` в промисе `play()`. Сейчас у тебя `.catch()` есть, а `.then()` нет, поэтому фолбэк никогда не убирается и двоит картинку под видео.
