## 🚀 Project Structure

Inside of your Astro project, you'll see the following folders and files:

```text
/
├── public/
├── src/
│   └── pages/
│       └── index.astro
└── package.json
```

Astro looks for `.astro` or `.md` files in the `src/pages/` directory. Each page is exposed as a route based on its file name.

There's nothing special about `src/components/`, but that's where we like to put any Astro/React/Vue/Svelte/Preact components.

Any static assets, like images, can be placed in the `public/` directory.

## 🧞 Commands

All commands are run from the root of the project, from a terminal:

| Command                   | Action                                           |
| :------------------------ | :----------------------------------------------- |
| `npm install`             | Installs dependencies                            |
| `npm run dev`             | Starts local dev server at `localhost:4321`      |
| `npm run build`           | Build your production site to `./dist/`          |
| `npm run preview`         | Preview your build locally, before deploying     |
| `npm run astro ...`       | Run CLI commands like `astro add`, `astro check` |
| `npm run astro -- --help` | Get help using the Astro CLI                     |

### 🚨 КРИТИЧЕСКИЕ ОШИБКИ (нужно исправить немедленно — это отпугивает клиентов)

**1. Placeholder-текст виден живым посетителям**
На странице **Услуги** в секции с фотографиями работ написано открытым текстом:
> "Реальные объекты нашей бригады — **замените фото на свои когда будут готовы**."

Это черновой текст разработчика, который случайно попал на продакшн. Любой потенциальный клиент прочитает это и решит, что компания ненастоящая или сайт не готов. **Удалить или заменить немедленно.**

**2. Пустая секция "Три причины выбрать ProAcoperis.md"**
На странице **"О компании"** есть заголовок раздела "ГАРАНТИИ — Три причины выбрать ProAcoperis.md", после которого идёт **совершенно пустой блок** — около половины экрана белого пространства. Контент там либо не добавлен, либо не рендерится. Клиент видит пустоту вместо аргументов.

**3. Сломанное/отсутствующее фото на главной**
В разделе "Последние реализованные проекты" на главной странице одна из трёх карточек **полностью пустая** — белый прямоугольник без изображения. Выглядит как технический сбой и разрушает доверие.

**4. Невидимый CTA-блок на странице "Наши работы"**
В самом конце страницы "Наши работы" финальная секция с призывом к действию и кнопкой практически **не читается** — тёмный текст на тёмном фоне, кнопка почти сливается с фоном. Весь потенциал этой страницы теряется.

**5. Ненастоящий номер телефона**
На всём сайте указан номер **+373 60 000 000** — это явный тестовый placeholder. Если это не исправлено для реального запуска, любой позвонивший клиент потерян навсегда. Это самый критичный элемент из всех.

### ⚠️ ПРОБЛЕМЫ ДИЗАЙНА И UX (стоит улучшить)

**Видеоблок — нет превью.** И на главной, и на странице "Наши работы" есть два видео. Они отображаются как чёрные прямоугольники с кнопкой play в самом углу. Без яркого превью-кадра (thumbnail) большинство пользователей не поймут, что это видео, и проскроллят мимо.

**Страница "Наши работы" — нет подписей к фото.** В разделе "Все объекты" показаны 6 фотографий, но ни у одной нет названия объекта, типа работы или года. Клиент не понимает, что именно видит — это беседка или жилой дом? Новая крыша или ремонт? Добавление коротких подписей (тип объекта, площадь, год) резко повысило бы доверие.

**Отсутствуют отзывы клиентов.** Ни на одной странице сайта нет раздела с отзывами или оценками. Это огромный пробел — отзывы реальных клиентов являются ключевым фактором выбора в строительной нише. Хотя бы 3–5 отзывов с именем, городом и фото объекта сильно увеличили бы конверсию.

**Кнопка "Бесплатный замер за 24 часа" на главной** разбита на две строки, выглядит неаккуратно по сравнению с соседней кнопкой "Наши работы".

**Логотип партнёра Технониколь.** На странице "О компании" есть блок "Официальный партнёр Технониколь", но изображение логотипа очень слабо видно — иконка мелкая и блёклая. Это важный аргумент доверия, он должен быть заметным.

**Страница "О компании" — "Вехи компании".** Хронология (2015, 2018...) показывает только 2 события, с очень большими пустыми промежутками между ними. Либо нужно добавить больше вех, либо сократить пустые отступы — сейчас это выглядит как незаконченная страница.

**FAQ на странице услуг.** Аккордеон с вопросами выглядит хорошо, но вопросы закрыты по умолчанию. Стоит хотя бы первый вопрос ("Сколько стоит выезд инженера?") открыть по умолчанию — это самый частый вопрос и сразу даст ответ.
